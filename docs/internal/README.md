# Internal Docs

This folder is the **historical build record** for Brahmaand: the up-front design blueprint, phase-by-phase plans, research notes, and the design decisions, roadmaps, and commercial/scaling thinking that shaped the project. It is kept public for transparency and reference.

These are **not** user-facing docs. If you just want to use or install Brahmaand, head back to the [project root README](../../README.md) and the guides in [`docs/`](../) ([iOS](../IOS.md), [Android](../ANDROID.md), [usage & legal](../USAGE-AND-LEGAL.md)).

## Contents

### Sub-areas

| Area | What's inside |
| --- | --- |
| [`blueprint/`](./blueprint/) | The numbered up-front design blueprint (`00-vision` … `08-testing`): architecture, data sources, HiPS, the star-catalog pipeline, WebXR, performance, pitfalls, and testing. |
| [`plan/`](./plan/) | Phase-by-phase build plans (`PHASE-0` … `PHASE-8`) plus the agent instructions that drove the build. |
| [`research/`](./research/) | Raw research notes behind the design: HEALPix math, HiPS format, Gaia/TAP pipelines, star rendering, Three.js/WebXR, LSST/Rubin, performance, and prior art. |
